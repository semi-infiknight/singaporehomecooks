## 1. Executive Summary (30 seconds)

Singapore Home Cooks is a **Turborepo monorepo** for a two-sided marketplace (home cooks ↔ customers) in Singapore. The product is **locally runnable end-to-end** with rich mocks and an **optional real Medusa backend** on `:9000`.

| Layer | Status | Notes |
|-------|--------|-------|
| **Mobile Customer** (`apps/mobile-customer`) | ✅ Full UX | Email/password auth; Medusa required |
| **Mobile Cook** (`apps/mobile-cook`) | ✅ Full UX | Separate app; cook login only |
| **Web** (Next.js `:3001`) | ✅ Customer only | Peach Comfort brand; `/login`; no dev switcher |
| **Medusa API** (`:9000`) | ✅ ~70% routes | Custom `/store/shc/*` + `/admin/shc/*`; admin UI at `/app` |
| **Auth (JWT)** | 🟡 Dev-ready | Customer: Medusa email/pass; Cook: SHC JWT; no dev switcher |
| **PayNow / PayU** | 🟡 Simulated | Manual ops confirm via admin route |
| **Production deploy** | ❌ Not live | Railway/EAS configs exist; founder keys needed |

**Do not trust `STATUS.md` alone** — it predates the 2026-06-14 Medusa wiring wave. **This file is the accurate integration snapshot.**

## 3. Integration Mode (critical for agents)

### Real Medusa toggle

Both clients use `@shc/api-client` (`packages/shc-api-client`):

- **Default:** real Medusa **ON** when env says so (not explicitly `false`)
- **Fallback:** if API fails, silently uses `mock-service` (same rules, in-memory)
- **Actor headers:** `x-shc-customer-id`, `x-shc-cook-id` (maps cook dev user → `cook_rose_tampines_001`)

| App | Env file | Key vars |
|-----|----------|----------|
| Mobile | `apps/mobile/.env.local` | `EXPO_PUBLIC_USE_REAL_MEDUSA=true`, `EXPO_PUBLIC_MEDUSA_BASE`, `EXPO_PUBLIC_MEDUSA_PUBLISHABLE_KEY` |
| Web | `apps/web/.env.local` | `NEXT_PUBLIC_USE_REAL_MEDUSA=true`, `NEXT_PUBLIC_SHC_API_BASE`, `NEXT_PUBLIC_MEDUSA_PUBLISHABLE_KEY` |

Generated by: `pnpm bootstrap:medusa` (requires Medusa healthy on `:9000`)

## 8. Known Gotchas (read before debugging)

1. **`pnpm medusa:start` disables admin** — use `pnpm medusa:dev:admin` for dashboard.
2. **Admin login fails on `127.0.0.1`** if `backendUrl` is hardcoded to `localhost` — fixed via `backendUrl: "/"` in config.
3. **Product ID mismatch:** Canonical seeds use `dish_*` IDs; Medusa DB seed uses `prod_*` — `getProduct` route has fuzzy fallback; prefer running `pnpm seed` in medusa.
4. **Cook dev user ID:** Mock login uses `cook_rose_tampines_001` (aligned with DB); `@shc/api-client` maps legacy `cook_auntierose`.
5. **Server cart is in-memory** — resets on Medusa restart; not production-ready.
6. **Auth is not enforced** on store routes — actor IDs come from headers (dev only).
7. **Large uncommitted diff** — see §10; do not assume `main` is clean.
