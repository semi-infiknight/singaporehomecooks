9. Platform Policies
9.1 Tiered Occasion Guarantee — Replacing New Cook Guarantee
Following four-perspective critical analysis, the New Cook Guarantee (flat S$10–15 credit for first 10 orders) is replaced by a Tiered Occasion Guarantee proportional to order value. This is legally cleaner (not a quality warranty — a goodwill commitment for objective failures) and more meaningful for high-value occasion orders.

Order Value
Guarantee for Verified Objective Failures
Under S$150
Platform credit S$10–15. Funded from platform commission. Cook earnings never reduced.
S$150 and above
Refund up to 50% of order value, capped at S$100. Funded from Guarantee Reserve (0.5% of high-value order GMV set aside monthly).

	•	Qualifying objective failures (both tiers): food not delivered at all, completely wrong dish delivered, verified food safety issue. Photo evidence required within 2 hours of collection. Subjective dissatisfaction never qualifies.
	•	Cook earnings are NEVER reduced by any guarantee payment. Guarantee funded entirely from platform commission revenue and the Guarantee Reserve. This is explicit in the Cook Agreement in English and Mandarin.
	•	New Cook badge: retained for cooks with fewer than 10 completed orders as a transparency indicator. Not the guarantee mechanism — the tiered guarantee applies to all cooks.
	•	Guarantee Reserve: 0.5% of GMV from orders above S$150, set aside monthly. Balance shown in CEO dashboard as Guarantee Reserve Liability (Account 2005). Insurer informed and coverage confirmed.

9.2 Minimum Order Value
S$50 minimum order value across the board. Single threshold — no tiered approach. Stored in platform_config (minimum_order_value: 50) — fully adjustable by CEO via admin interface without any code change.
	•	Applies to all standard listings and order types. Enforced at checkout — if order total falls below S$50, customer sees: 'Minimum order is S$50. Please add more portions or select a different listing.'
	•	Tasting portions (maximum S$8) are exempt — flagged as tasting_portion: true in database, minimum order check bypassed.
	•	Commission tier impact: with S$50 minimum, effective tiers are S$50–S$80 at 18%, S$81–S$150 at 15%, above S$150 at 12%. The 20% tier (up to S$30) is dormant at standard minimum.
	•	Adjustable examples: raise to S$80 during CNY/Hari Raya peak. Drop to S$30 for promotional campaigns. All via platform_config admin panel, zero developer involvement.

11B.1 Document Numbering — Corrected Design
Order numbers and receipt numbers use SEPARATE sequential counters. This ensures no gaps in the receipt sequence — a clean requirement for IRAS and any future audit.

11B.2 Commission Transparency Decision
Customer pays the listed price. Commission is deducted from the cook's share — not added on top. Receipt shows the split transparently: 'Amount to [Cook Name]: S$19.68 / Platform service fee (18%): S$4.32 / Total charged: S$24.00'. Transparency about the commission builds trust. Opacity creates resentment.

CRITICAL PRE-LAUNCH ACTION: Engage Singapore tax specialist to determine whether platform's taxable turnover is (A) gross GMV (principal basis — early registration) or (B) commission revenue only (agent basis — later registration). 	•	5-year accounting record retention: satisfied by double-entry ledger and receipt retention policy.

Phase 2 — At the Threshold
	•	CEO dashboard 75% flag triggers: accountant engaged immediately, GST registration filed with IRAS through myTax Portal. Processing: 2–4 weeks.

Phase 3 — After Registration
	•	Price display: all prices must be GST-INCLUSIVE (Singapore law). Cook sets GST-exclusive price. Platform calculates and displays GST-inclusive price. When GST rate changes: update gst_rate in config → all displayed prices updated automatically, no code change.
	•	Tax invoices: receipts become TAX INVOICES. Must include GST registration number, GST rate, GST amount separately, GST-exclusive amount, GST-inclusive total. Simplified tax invoice (no buyer details) for transactions under S$1,000 — most food orders qualify.
	•	Quarterly F5 return: Xero handles preparation when correctly configured. Accountant reviews and submits. Due: last day of month following quarter end.
	•	Input tax claims: GST paid on Xero subscription, legal fees, insurance (from GST-registered suppliers) is claimable — partially offsets compliance cost.
	•	GST on Claude API (Overseas Vendor Registration): Anthropic is a US company — check if GST-registered in Singapore. If not: platform may need to self-account for GST on API costs under reverse charge mechanism. Accountant to advise. At S$30–100/month API cost, amount is trivial but compliance obligation must be correctly handled.

16B. Comprehensive AI Integration Strategy
Every AI feature on Singapore Home Cooks runs through a single provider — the Claude API by Anthropic. One interface, one billing account, one quality standard. No patchwork of different AI providers. This simplifies development and ensures consistency across all 18 features. Estimated cost: S$30–100 per month at launch scale.

16B.1 Architecture — RAG for Chatbots, Direct API for Features
	•	All chatbots use Retrieval-Augmented Generation (RAG): Claude reads the platform's specific knowledge base stored in Supabase and answers only from documented content. Never makes up answers. Knowledge base updated via simple admin interface — no retraining required.
	•	All other AI features use direct Claude API calls with feature-specific system prompts.
	•	Standard pattern: claude-sonnet-4-20250514, max_tokens 1000, feature system prompt, user input or platform data.
	•	All responses wrapped in try-catch. JSON responses strip markdown fences before parsing. On failure: graceful fallback — empty structured form, generic message, or route to human. Never show technical errors to users.
	•	Feedback loop: thumbs up/down on every chatbot response. Thumbs down reviewed weekly. Knowledge base updated. Chatbot improves continuously without model retraining.

16C. Platform Flexibility & Agility Architecture
Two architectural principles designed in from day one: a configurable commission engine where every rule lives in a database table, and an agile platform architecture where new features can be added without touching existing code.

16C.1 Configurable Commission Engine
Commission rules live in a database table — never in code. Code reads and applies rules. Zero code changes, zero redeployments needed to change any commission rate or add any new commission rule. All changes made through CEO dashboard admin interface.

16C.2 Five Agile Architectural Principles
	•	Feature flags: every new feature launched behind a flag. Toggle on/off without code deployment. CEO dashboard feature flags panel. Test with specific users before full rollout.
	•	Plugin-style modules: each feature has own database tables, API routes, and components. Adding a feature adds a module — existing code is never touched.
	•	Event-driven automation: actions fire events (order.collected, cook.registered, etc.). New automations simply listen — event-firing code is never modified.
	•	Configuration tables: every value that might change lives in platform_config. Change one row, effect is immediate. No redeployment.
	•	Migration discipline: all schema changes via Supabase migrations. New features only add tables — existing data never at risk.
	•	AI prompt management: prompts stored in database (ai_prompts table). Update any AI feature through admin interface. Version history retained. A/B test prompts without code.

No more managing orders on WhatsApp. We handle everything — orders, payments, and customer discovery. We even write your listing description for you from your ingredients. And we pay for your food safety certification. No upfront cost — we only earn when you earn. I'd love to have you as a Founding Cook.'

Recruitment Channels
	•	Facebook community groups: Singapore Home Bakers, Mummies@SG, SG Home Cooks and Bakers, SG Malay Cooking, Singapore Indian Cooking, HDB estate groups.
	•	Instagram direct outreach: personalised DMs referencing specific food by name. #sgbaker #sgcakes #sghomecook.
	•	WhatsApp community networks, RC/CC networks, CC cooking class instructors.
	•	Word of mouth from founding cooks — most cost-effective channel, cannot be manufactured.

19.2 Customer Acquisition
	•	Cook network spillover (primary early channel). Cook Spotlight organic reach.
	•	Home Credits referral programme (Phase 3): S$4 cost per referred customer acquired.
	•	Instagram and TikTok: Recipe Story format — two sentences on recipe origin, dish photo, ingredient list.
	•	SEO: individual indexed cook and listing pages rank for Singapore food search terms.
	•	Paid social: S$500/month from Month 3. Optimise for cost per first order.
	•	LinkedIn: platform story and occasion food content for HR managers and office admins.
